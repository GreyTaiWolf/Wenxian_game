# 策划数值文档

本文档记录当前版本的数值基线、公式和调参记录。任何修改境界、属性、技能、敌人、奖励、成本、概率、掉落或商店价格时，都必须同步更新本文档。

## 数值原则

- 早期成长应稳定、短反馈，玩家每次聚气或战斗都能看到收益。
- 突破成本用于引导玩家探索不同玩法，不应只依赖重复聚气。
- 新资源必须有明确来源和用途，避免只产出不消耗。
- 战斗难度应按区域和境界递进，避免普通野外战斗超过同阶段玩家承受能力。
- 关键数值优先放在 `src/data` 或 `src/game`，不要散落在 UI 组件中。

## 战斗公式

来源：`src/game/combatEngine.ts`

- 行动顺序：所有存活单位按速度从高到低排序。
- 基础伤害：`floor(攻击 * 技能威力 - 目标防御 * 防御系数)`，最低为 `1`。
- 普攻和撕咬防御系数：`0.45`。
- 其他伤害技能防御系数：`0.35`。
- 闪避：伤害技能结算前按目标闪避率判定；成功时本次伤害完全免疫，闪避率结算上限为 `75%`。
- 暴击：命中暴击率时，伤害乘以 `1.5`。
- 防御状态：受到伤害乘以 `0.5`。
- 护主减伤：受到伤害乘以 `0.65`。
- 攻击下降：攻击乘以 `0.85`。
- 回春诀治疗：`max(12, floor(28 + 神识 * 0.6))`。
- 回灵效果：`max(5, floor(10 + 神识 * 0.3))`，当前技能表暂无玩家可用回灵技能。
- 逃跑成功率：`72%`。
- 玩家战斗携带容量：`2` 个修士技能、`1` 个神通槽、`1` 个丹药槽。
- 普攻不占用携带槽，也不在 UI 中显示；无可用携带行动时可作为随机敌方单体的兜底动作。

## 调参记录

新增数值改动时，在本节最上方追加记录。

### 2026-05-18：速度命名与闪避属性
- 改动项：玩家可见敏捷属性统一命名为“速度”，新增真实战斗属性“闪避”。
- 旧值：角色和敌人只有速度与暴击；布履只提供速度 +1；伤害技能不会被完全闪避。
- 新值：主角初始闪避 0.05，突破每次 +0.01；布履提供速度 +1、闪避 +1%；敌人和队友拥有独立闪避值，伤害技能命中前按目标闪避率判定，结算上限 75%。
- 原因：让敏捷类属性更清晰，并为后续装备、灵宠和速度型敌人提供新的防御向成长空间。
- 影响范围：玩家属性、装备加成、敌人模板、队友模板、战斗伤害公式、旧存档兼容和背包属性显示。
- 验证方式：TypeScript 检查、Vite build、浏览器验证属性显示、旧存档加载、战斗闪避日志和装备加成。

### 2026-05-18：南疆开放数值基线
