# 策划数值文档

本文档记录当前版本的数值基线、公式和调参记录。任何修改境界、属性、技能、敌人、奖励、成本、概率、掉落或商店价格时，都必须同步更新本文档。

## 数值原则

- 早期成长应稳定、短反馈，玩家每次聚气或战斗都能看到收益。
- 突破成本用于引导玩家探索不同玩法，不应只依赖重复聚气。
- 新资源必须有明确来源和用途，避免只产出不消耗。
- 战斗难度应按区域和境界递进，避免普通野外战斗超过同阶段玩家承受能力。
- 关键数值优先放在 `src/data` 或 `src/game`，不要散落在 UI 组件中。

## 战斗公式

来源：`src/game/combatEngine.ts`

- 行动顺序：所有存活单位按速度从高到低排序。
- 基础伤害：`floor(攻击 * 技能威力 - 目标防御 * 防御系数)`，最低为 `1`。
- 普攻和撕咬防御系数：`0.45`。
- 其他伤害技能防御系数：`0.35`。
- 闪避：伤害技能结算前按目标闪避率判定；成功时本次伤害完全免疫，闪避率结算上限为 `75%`。
- 暴击：命中暴击率时，伤害乘以 `1.5`。
- 防御状态：受到伤害乘以 `0.5`。
- 护主减伤：受到伤害乘以 `0.65`。
- 攻击下降：攻击乘以 `0.85`。
- 回春诀治疗：`max(12, floor(28 + 神识 * 0.6))`。
- 回灵效果：`max(5, floor(10 + 神识 * 0.3))`，当前技能表暂无玩家可用回灵技能。
- 逃跑成功率：`72%`。
- 玩家战斗携带容量：`2` 个武技槽、`2` 个功法槽、`1` 个神通槽、`1` 个法宝槽、`1` 个丹药槽。
- 功法槽当前仅作为 UI 预留，暂不产生主动战斗效果。
- 法宝槽当前读取背包装备中的法宝栏，第一版仅展示装备状态，暂不产生主动战斗效果。
- 普攻不占用携带槽，也不在 UI 中显示；无可用携带行动时可作为随机敌方单体的兜底动作。

## 调参记录

新增数值改动时，在本节最上方追加记录。

### 2026-05-18：洞府闭关与聚灵阵
- 改动项：新增洞府闭关结算、聚灵阵等级、闭关倍率、最长闭关时间和升级成本。
- 旧值：洞府只显示闭关效率、最长闭关和聚灵阵加成占位；实际离线结算未实现。
- 新值：基础闭关效率为 `12 修为 / 分钟`；0/1/2/3 级聚灵阵倍率为 `100%/110%/122%/135%`，最长闭关为 `8/10/12/14` 小时；领取收益不超过当前境界剩余修为。
- 原因：让筑基后洞府成为真实长期成长系统，并消耗中州、南疆历练材料。
- 影响范围：洞府存档字段、修为结算、背包材料消耗、聚灵阵升级、旧存档兼容和洞府 UI。
- 验证方式：TypeScript 检查、Vite build、旧存档加载、开始闭关、领取收益、修为满提示、聚灵阵升级和资源不足日志。

### 2026-05-18：单物品品级倍率与装备特效
- 改动项：移除六品独立物品 ID 和背包样品补齐，改为每个正式物品通过 `grade` 调用六品倍率、词条库和 UI 特效库。
- 旧值：每个物品家族生成凡品、良品、珍品、玄品、地品、天品六个版本，例如 `healing_powder_common` 到 `healing_powder_heaven`。
