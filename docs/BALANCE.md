# 策划数值文档

本文档记录当前版本的数值基线、公式和调参记录。任何修改境界、属性、技能、敌人、奖励、成本、概率、掉落或商店价格时，都必须同步更新本文档。

## 数值原则

- 早期成长应稳定、短反馈，玩家每次聚气或战斗都能看到收益。
- 突破成本用于引导玩家探索不同玩法，不应只依赖重复聚气。
- 新资源必须有明确来源和用途，避免只产出不消耗。
- 战斗难度应按区域和境界递进，避免普通野外战斗超过同阶段玩家承受能力。
- 关键数值优先放在 `src/data` 或 `src/game`，不要散落在 UI 组件中。

战力实时派生，不再保存为真实数值：

```ts
rawStatScore =
  maxHp * 0.10 +
  maxSpirit * 0.15 +
  attack * 1.80 +
  defense * 1.60 +
  speed * 1.20 +
  90 * critRate * critDamage +
  120 * dodgeRate +
  1.20 * spiritSense;

power = Math.round(rawStatScore * realmCoef[majorRealm] * stageCoef[phase]);
```

闪避在装备 V1 中正式启用，最终闪避率硬上限为 35%。神识字段统一为 `spiritSense`，元婴前装备不会生成神识主属性或词条。

突破成功改为套用目标境界基础属性：`hp/spirit` 回满，寿元更新，心境 `+5`，清空灵气不稳层数。

突破成功率：

```ts
breakRatePct = clamp(
  baseRate +
  (mindValue - 50) * 0.35 +
  (comprehension - 10) * 1.20 -
  injuryStacks * 3 -
  instabilityStacks * 4 -
  agePenalty,
  5,
  95
);

agePenalty = Math.max(0, age / lifespan - 0.80) * 25;
```

突破失败：

- 小境界失败：扣除突破成本；修为回落到当前上限 `88%`；气血/灵力降至 `60%`；心境 `-6`；灵气不稳 `+1`。
- 大境界失败：扣除突破成本与关键丹药；修为回落到当前上限 `75%`；气血/灵力降至 `30%`；心境 `-12`；灵气不稳 `+2`；伤势 `+1`。

## 战斗公式

来源：`src/game/combatEngine.ts`

- 行动顺序：所有存活单位按速度从高到低排序，先机和借势等临时速度提高会参与排序。
- 基础伤害：`floor(攻击 * 技能威力 - 目标防御 * 防御系数)`，最低为 `1`。
- 普攻和撕咬防御系数：`0.45`。
- 其他伤害技能防御系数：`0.35`。
- 闪避：普攻、撕咬、扑击和引气斩吃完整闪避；飞叶术吃 50% 闪避；兽吼等范围伤害不可闪避。最终闪避软上限 30%，硬上限 35%。
- 暴击：命中暴击率时，伤害乘以角色自身暴击倍率，当前基础为 `1.50`；暴击率软上限 35%、硬上限 50%，暴伤普通上限 2.20，仙/神专属可到 2.50。
- 防御状态：受到伤害乘以 `0.5`。
- 护主减伤：受到伤害乘以 `0.65`。
- 攻击下降：攻击乘以 `0.85`。
- 回春诀治疗：`max(12, floor(28 + spiritSense * 0.6))`。
- 回灵效果：`max(5, floor(10 + spiritSense * 0.3))`；装备回灵词条可在回合开始额外恢复灵力。
- 逃跑基础成功率：`72%`，遁走词条可提高，上限 95%。
- 玩家战斗携带容量：`2` 个武技槽、`2` 个功法槽、`1` 个神通槽、`1` 个法宝槽、`1` 个丹药槽。
- 功法槽当前仅作为 UI 预留，暂不产生主动战斗效果。
- 法宝槽读取 `artifact` 装备；存在主动、领域或法则词条时可催动，否则展示被动词条状态。
- 普攻不占用携带槽，也不在 UI 中显示；无可用携带行动时可作为随机敌方单体的兜底动作。

## 时间推进与年龄增长（2026-05）

- 历法常量：
  - `CALENDAR_MONTHS_PER_YEAR = 12`
  - `CALENDAR_DAYS_PER_MONTH = 30`
  - `CALENDAR_DAYS_PER_WEEK = 7`
- 移动耗时配置：
  - 大世界每移动 1 格：`WORLD_TILE_MOVE_DAYS = 7`
  - 州域每移动 1 格：`REGION_TILE_MOVE_DAYS = 1`
- 年龄增长公式（按天累计）：
  - `age += days / (12 * 30)`
  - 显示保留两位小数。
- 该年龄继续参与原有突破年龄惩罚（`age / lifespan`）计算，因此远行与寿元压力形成策略联动。
